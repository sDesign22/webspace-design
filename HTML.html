HTML is the World Wide Web's core (standard) markup language for creating Web pages, it describes the structure of a Web page and consists of a series of elements. HTML was primarily designed as a language for semantically describing scientific documents. Its design has enabled it to be adapted over subsequent years, to describe a number of other types of documents and even applications. These definitions allow HTML processors (such as web browsers or search engines), to present and use documents or applications in a wide variety of contexts. For example, Search engines can use headings, to effectively index a page or provide links to subsections of the page from results.
HTML Elements tell the browser how to display content, and label content using start and end tags: <Start-Tag> Content is everything inside of the start and end tags. </End-Tag>
Some Elements are Empty Elements and do not have an End Tag: <Empty-Element>
HTML documents might be rendered to a screen, through a speech synthesizer or on a braille display. To influence exactly how such rendering takes place, authors can also use a styling language such as CSS.

HTML Editors

A text editor is all you need to learn HTML! Web pages can be created and modified using professional HTML editors, however for learning; it's recommend to use a simple text editor like Notepad (PC) or TextEdit (Mac).


You save the file on your computer, using either .htm or .html as the file extension (there is no difference). And set the encoding to UTF-8 (Unicode), witch is the preferred encoding for HTML; it is also the default character set, if one is not specified otherwise. To view the HTML Page in your Browser, double click on the file, or right-click it and choose "Open with".

Web Browsers

The purpose of a web browser (such as Chrome, Edge, Firefox or Safari) is to read HTML documents and display them correctly. A browser does not display the HTML tags, but uses them to determine how to display the document; then parse this markup, turning it into a DOM (Document Object Model) tree. A DOM tree is an in-memory representation of a document, the HTML DOM is an API (Programming Interface) for JavaScript. DOM trees contain several kinds of nodes, in particular a DocumentType node, Element nodes, Text nodes, Comment nodes, and in some cases Processing Instruction nodes. Each element in the DOM tree is represented by an object, and these objects have APIs so that they can be manipulated from scripts in the page. Scripts (typically in JavaScript) are small programs that can be embedded using the <script> element or using event handler content attributes.

Inspect an HTML Element

To view HTML Source Code, Right-click in an HTML page and select "View Page Source" in Chrome or "View Source" in Edge (or similar methods in other browsers). Or to Inspect an HTML Element, Right-click on an element (or a blank area), and choose "Inspect" or "Inspect Element" to see what the elements are made up of. Doing any of these this will open a window containing the HTML source code of the page, you can then also edit the HTML on-the-fly in the Elements panel that opens.

Whitespace

With HTML, you cannot change the display by adding extra spaces or extra lines in your HTML code. The browser will automatically remove any extra spaces and lines when the page is displayed.

HTML Character Sets (Encoding)

To display an HTML page correctly, a web browser must know which character set to use, the encoding UTF-8 (Unicode); is the default character set. Unicode is a character set, and UTF-8 is encoding; Unicode is a list of characters with unique decimal numbers (code points). Encoding is how these numbers are translated into binary numbers, to then be stored in a computer, UTF-8 encoding will store "hello" with binary. So the encoding translates numbers into binary, while the character set translates characters into numbers.
The character encoding for the early web was ASCII, to standardize storing of text, the American Standard Code for Information Interchange (ASCII) was created. It defined a unique binary number for each storable character to support the numbers from 0-9, the upper and lower case alphabet and special characters (like ! $ + - ( @ > , .). Computer data is stored as binary codes (01000101) in the electronics.

The character set most often used in HTML 4 was ISO-8859-1, witch is an extension to ASCII, with added international characters. However, when a browser detects ISO-8859-1, it normally defaults to Windows-1252, because Windows-1252 has 32 more international characters.
The Unicode Consortium then developed the UTF-8 and UTF-16 standards, because the ISO-8859 character-sets are limited and not a compatible multilingual environment. The Unicode Standard covers almost all of the characters, punctuations, and symbols in the world; it has become a success, implemented in HTML, XML, Java, JavaScript, E-mail, ASP, PHP, etc. The Unicode standard is also supported in many operating systems and in all modern browsers. The Unicode Consortium cooperates with the leading standards development organizations, like ISO, W3C and ECMA.

A character-set different from UTF-8 can be specified in the <meta> tag: <meta charset="ISO-8859-1"> - HTML4 supports only the UTF-8 encoding, while HTML5 processors support both UTF-8, UTF-16 and the Windows-1252 or ISO-8859 character sets.

HTML Reserved Characters & Entities

Some characters are reserved in HTML, Reserved Characters must be replaced with Character Entities; witch is done using the entities name or number: &entity_name; | &#entity_number;
If you use the less "<" or greater than ">" signs, the browser might mix them with tags; character entities are used to display thease characters. Another common use is the non-breaking space, it is to prevent browsers from truncating spaces in HTML pages. If you write 10 spaces in text, the browser will remove 9 of them, so to add spaces to the text, you can use a character entity. Browsers may not support all entity names, but the support for entity numbers is good.

Emojis in HTML

Emojis look like images or icons, but are not; they are characters from the UTF-8 character set. Many UTF-8 characters cannot be typed on keyboard, but displayed using numbers called entity numbers. To let the browser understand you are displaying a character, you must start the entity number with & and #, then end with a semicolon (;). Since Emojis are characters, they can be copied, displayed and sized just like any other character in HTML.

Uniform Resource Locator (URL) Encoding

A Uniform Resource Locator (URL) is used to address a document or other data on the web, it is another word for a web address; browsers request pages from web servers by using a URL. A URL can be composed of words like w3schools.com, or as an Internet Protocol (IP) address like 192.68.20.50. Most people, however, enter the name when surfing, because names are easier to remember than numbers.
URL encoding (or Percent Encoding) converts characters into a format that can be transmitted over the Internet, URLs can only be sent over the Internet using the ASCII character-set. Since URLs often contain characters outside the ASCII set, the URL has to be converted into a valid ASCII format. The URL encoding replaces unsafe ASCII characters with a "%" followed by two hexadecimal digits. URLs cannot contain spaces, URL encoding normally replaces a space with a plus (+) sign or with %20.

A web address such as https://www.w3schools.com/html/default.asp follows these syntax rules: scheme://prefix.domain:port/path/filename

  • scheme - Defines the type of Internet service, the most common is http or https.
    This table lists some common schemes:
    Scheme Used for
    http - HyperText Transfer Protocol Common web pages - Is not encrypted
    https - Secure HyperText Transfer Protocol Secure web pages - Is encrypted
    ftp - File Transfer Protocol Downloading or uploading files
    file A file on your computer
  • prefix - Defines a domain prefix, the default for http is www
  • domain - Defines the Internet domain name, such as w3schools.com
  • port - Defines the port number at the host, the default for http is 80
  • path - Defines a path at the server, and if omitted, the root directory of the site
  • filename - Defines the name of a document or resource

Hyper Text Transfer Protocol (HTTP)

The World Wide Web (WWW) is about communication between web clients and servers, this is done by sending HTTP Requests and receiving HTTP Responses. Clients are often browsers (Chrome, Edge, Safari), but they can be any type of program or device, Servers are most often computers in the cloud.
The communication is done by request and response methods, shown below (read more on HTTP Methods):

  1. A client (a browser) sends an HTTP request to the web
  2. A web server receives the request
  3. The server runs an application to process the request
  4. The server returns an HTTP response (the output) to the browser
  5. The client (the browser) receives the response
HTTP Status Messages - The server always returns a message for every request, these messages are called HTTP Status Messages (See a list of HTTP Status Messages that might be returned). When a browser requests from a web server, an error may also occur and the server might return an error code like "404 Not Found"; it's common to name these HTML error messages.
